# 주문 취소 정책

## 자동 취소 가능 조건

앱에서 고객이 직접 취소할 수 있는 조건입니다.

- 주문 상태가 **CREATED** (주문 접수 직후)
- 주문 상태가 **ACCEPTED** (가게가 접수 확인은 했으나 조리 시작 전)

이 상태에서는 앱의 "주문 취소" 버튼으로 즉시 취소되며, 결제 금액은 전액 환불됩니다.

## 자동 취소 불가 조건

- **COOKING** (조리 시작): 조리가 시작된 이후에는 자동 취소가 불가합니다. 상담원 연결이 필요합니다.
- **DELIVERING** (배달 중): 배달이 시작된 이후에는 수령 후 반품 절차를 따릅니다.
- **DELIVERED** (배달 완료): 별도의 환불 정책(배달 완료 후 환불)을 따릅니다.

## 취소 요청 시 안내 문구

- 자동 취소 가능 상태: "취소가 완료되었습니다. 결제 취소는 카드사에 따라 최대 7영업일이 소요될 수 있습니다."
- 자동 취소 불가 상태: "조리가 이미 시작되어 자동 취소가 어렵습니다. 상담원 연결로 도와드리겠습니다."

## 중복 취소 요청 (멱등성)

이미 취소된 주문에 대해 다시 취소 요청이 들어와도 에러로 응답하지 않고,
"해당 주문은 이미 취소된 상태입니다"로 안내합니다.

## 고객 귀책 취소

- 고객이 주소를 잘못 입력해 반송된 경우: 배달비는 환불 대상이 아닙니다.
- 고객이 수령을 거부한 경우: 음식값 전액 + 배달비 왕복 비용이 발생할 수 있습니다.
